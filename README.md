# Redis可视化管理工具

一个基于Python tkinter开发的Redis图形化管理工具，支持多连接管理、Key-Value编辑、服务状态查看等功能。

## 功能特性

### 1. 多Redis连接管理
- ✅ 添加/编辑/删除多个Redis连接配置
- ✅ 连接信息本地持久化（JSON文件）
- ✅ 一键切换不同Redis连接
- ✅ 实时显示连接状态

### 2. Key-Value 管理
- ✅ 展示当前库所有key（支持分页、搜索过滤）
- ✅ 支持5种核心数据类型：**string、hash、list、set、zset**
- ✅ 查看key详情：value内容、类型、过期时间
- ✅ 编辑/修改key-value
- ✅ 新增key（选择类型、设置值、过期时间）
- ✅ 删除key、重命名key、设置/移除过期时间
- ✅ 刷新key列表

### 3. Redis服务状态查看
- ✅ 实时展示Redis服务信息：版本、运行模式、内存使用、客户端数、key总数
- ✅ 清晰格式化展示INFO命令结果
- ✅ **查看客户端连接列表**：IP、端口、数据库、空闲时间等详细信息
- ✅ **主动断开指定客户端连接**：支持踢出异常或不需要的连接

### 4. 数据库操作
- ✅ 支持切换Redis数据库（0-15）
- ✅ 清空当前数据库所有key
- ✅ 清空所有数据库

## 使用说明

### 首次使用 - 添加连接

1. 启动程序后，点击顶部 **"管理连接"** 按钮
2. 在弹出的连接管理窗口中，点击 **"添加"** 按钮
3. 填写连接信息：
   - **别名**：给连接起一个易记的名字（如：本地开发、生产环境）
   - **主机**：Redis服务器地址（如：127.0.0.1）
   - **端口**：Redis端口（默认：6379）
   - **密码**：Redis密码（如果没有留空）
   - **数据库**：数据库编号（0-15）
4. 可以点击 **"测试连接"** 验证配置是否正确
5. 配置完成后关闭对话框

### 连接Redis

1. 从顶部状态栏的数据库下拉框选择数据库编号（0-15）
2. 程序会自动连接到Redis（如果没有连接，会提示先添加连接）

### 查看Key列表

- 中间区域显示当前数据库的所有key
- 支持搜索过滤：在搜索框输入关键词，点击"搜索"
- 支持分页：使用底部"上一页"/"下一页"按钮翻页

### 查看Key详情

- 在key列表中点击某个key，右侧会显示详细信息：
  - Key名称
  - 数据类型
  - 过期时间（TTL）
  - Value内容

### 新增Key

1. 点击左侧 **"➕ 新增Key"** 按钮
2. 填写Key名称
3. 选择数据类型（string/hash/list/set/zset）
4. 输入对应的值（不同类型有不同输入格式）
5. 设置过期时间（秒，0表示永不过期）
6. 点击"保存"

### 编辑Key

1. 在key列表中选择要编辑的key
2. 点击左侧 **"✏️ 编辑Key"** 按钮（或双击key）
3. 修改值后点击"保存"

### 删除Key

1. 在key列表中选择要删除的key
2. 点击左侧 **"🗑️ 删除Key"** 按钮
3. 确认删除操作

### 重命名Key

1. 在key列表中选择要重命名的key
2. 点击左侧 **"📝 重命名Key"** 按钮
3. 输入新的key名称

### 设置过期时间

1. 在key列表中选择key
2. 点击左侧 **"⏰ 设置过期时间"** 按钮
3. 输入过期时间（秒）
4. 输入0表示移除过期时间（永不过期）

### 清空数据库

- **清空当前库**：仅清空当前选中的数据库
- **清空所有库**：清空所有16个数据库（⚠️ 危险操作，请谨慎使用）

### 查看服务器信息

点击顶部 **"服务器信息"** 按钮，查看：
- Redis版本
- 运行模式（standalone/cluster/sentinel）
- 内存使用情况
- 客户端连接数
- key总数
- 其他统计信息

## 数据类型输入格式

### String
```
直接输入文本值
```

### Hash
```
格式：field=value（每行一个字段）
示例：
name=张三
age=25
email=test@example.com
```

### List
```
格式：每行一个元素
示例：
item1
item2
item3
```

### Set
```
格式：每行一个成员
示例：
member1
member2
member3
```

### ZSet（有序集合）
```
格式：member=score（每行一个成员，score为数字）
示例：
player1=100
player2=85.5
player3=92
```

## 注意事项

1. **首次运行**：需要先添加Redis连接配置才能使用功能
2. **密码安全**：连接密码会保存在本地配置文件中，请注意文件权限
3. **大数据量**：如果数据库中有大量key，搜索和刷新可能需要一些时间
4. **过期时间**：设置过期时间后，key到期会自动删除
5. **清空操作**：清空数据库是不可恢复的操作，请谨慎使用

## 常见问题

### Q: 连接失败怎么办？
A: 检查以下几点：
- Redis服务器是否启动
- 主机地址和端口是否正确
- 密码是否正确
- 防火墙是否阻止连接

### Q: 中文显示乱码？
A: 程序已设置UTF-8编码，如果仍有问题请确保系统支持中文显示。

### Q: 如何备份连接配置？
A: 连接配置保存在 `connections.json` 文件中，可以手动备份此文件。
