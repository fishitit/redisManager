# Redis可视化管理工具 - 打包指南

本文档说明如何将软件打包为Windows和macOS可执行文件。

---

## 一、打包环境要求

### 1.1 依赖安装
```bash
pip install redis pyinstaller
```

### 1.2 Python版本
- Python 3.6 或以上版本
- 推荐使用与目标系统相同的Python版本打包

---

## 二、macOS 打包

### 2.1 自动打包（推荐）

在项目根目录执行：
```bash
./build_mac.sh
```

**输出位置**：
```
dist/RedisVisualManager/          # 打包目录
dist/RedisVisualManager/RedisVisualManager  # 可执行文件
dist/RedisVisualManager.app/      # macOS应用包
```

### 2.2 手动打包

```bash
# 清理旧文件
rm -rf build dist *.spec __pycache__

# 执行打包
pyinstaller --name="RedisVisualManager" \
    --onedir \
    --windowed \
    --add-data "models:models" \
    --add-data "views:views" \
    --add-data "controllers:controllers" \
    --add-data "utils:utils" \
    main.py
```

### 2.3 打包参数说明

| 参数 | 说明 |
|------|------|
| `--name` | 输出文件名称 |
| `--onedir` | 目录模式（推荐），包含所有依赖文件 |
| `--windowed` | 无控制台窗口（GUI程序） |
| `--add-data` | 添加数据文件，格式：`源路径:目标路径` |

### 2.4 运行打包后的程序

**方式一：直接运行可执行文件**
```bash
cd dist/RedisVisualManager
./RedisVisualManager
```

**方式二：运行.app应用包**
```bash
open dist/RedisVisualManager.app
```
或双击 `RedisVisualManager.app`

## 三、Windows 打包

### 3.1 自动打包（推荐）

在项目根目录双击执行：
```
build_win.bat
```

或在命令行执行：
```cmd
build_win.bat
```

**输出位置**：
```
dist\RedisVisualManager\                    # 打包目录
dist\RedisVisualManager\RedisVisualManager.exe  # 可执行文件
```

### 3.2 手动打包

在Windows命令提示符或PowerShell中执行：
```cmd
REM 清理旧文件
rmdir /s /q build dist
del /q *.spec
rmdir /s /q __pycache__

REM 执行打包
pyinstaller --name="RedisVisualManager" ^
    --onedir ^
    --windowed ^
    --add-data "models;models" ^
    --add-data "views;views" ^
    --add-data "controllers;controllers" ^
    --add-data "utils;utils" ^
    main.py
```

### 3.3 Windows打包注意事项

**重要**：
1. **必须在Windows系统上打包**
   - macOS上无法打包Windows版本
   - 需要在Windows电脑上执行`build_win.bat`

2. **路径分隔符**：Windows使用分号`;`而非冒号`:`
   - 正确：`--add-data "models;models"`
   - 错误：`--add-data "models:models"`

3. **文件扩展名**：Windows输出应为`.exe`文件
   - 目录模式：`dist\RedisVisualManager\RedisVisualManager.exe`
   - 单文件模式：`dist\RedisVisualManager.exe`

4. **杀毒软件误报**：PyInstaller打包的程序可能被误报
   - 添加Windows Defender白名单
   - 或其他杀毒软件白名单

5. **Python环境**：
   - 确保安装Python时勾选"Add Python to PATH"
   - 建议使用Python 3.8-3.11版本

### 3.4 运行打包后的程序

双击 `dist\RedisVisualManager\RedisVisualManager.exe` 即可运行

## 四、单文件模式（可选）

如果希望打包为单个可执行文件，修改参数：

### macOS
```bash
pyinstaller --name="RedisVisualManager" \
    --onefile \
    --windowed \
    --add-data "models:models" \
    --add-data "views:views" \
    --add-data "controllers:controllers" \
    --add-data "utils:utils" \
    main.py
```

### Windows
```cmd
pyinstaller --name="RedisVisualManager" ^
    --onefile ^
    --windowed ^
    --add-data "models;models" ^
    --add-data "views;views" ^
    --add-data "controllers;controllers" ^
    --add-data "utils;utils" ^
    main.py
```

**注意**：
- `--onefile`模式启动较慢（需解压到临时目录）
- 推荐使用`--onedir`模式以获得更好的性能

## 六、打包后文件结构

### 6.1 onedir模式（推荐）

```
dist/RedisVisualManager/
├── RedisVisualManager          # 主程序（macOS）
├── RedisVisualManager.exe      # 主程序（Windows）
├── _internal/                  # 依赖文件
│   ├── base_library.zip
│   ├── libpython3.x.dylib      # Python运行时
│   ├── _tkinter*.so
│   └── ...其他依赖
└── 其他运行时文件
```

### 6.2 onefile模式

```
dist/
└── RedisVisualManager          # 单文件（macOS）
或
dist/
└── RedisVisualManager.exe      # 单文件（Windows）
```

## 七、分发说明

### 7.1 macOS分发

1. **压缩分发**
   ```bash
   cd dist
   zip -r RedisVisualManager-mac.zip RedisVisualManager/
   ```

2. **创建DMG镜像**
   ```bash
   hdiutil create -volname "RedisVisualManager" \
       -srcfolder RedisVisualManager \
       -ov -format UDZO RedisVisualManager-mac.dmg
   ```

3. **公证（可选）**
   - 需要Apple开发者账号
   - 使用`xcrun altool`进行公证

### 7.2 Windows分发

1. **压缩分发**
   ```cmd
   cd dist
   powershell Compress-Archive -Path RedisVisualManager -DestinationPath RedisVisualManager-win.zip
   ```

2. **创建安装程序（可选）**
   - 使用Inno Setup
   - 使用NSIS
   - 使用Advanced Installer

## 八、常见问题

### Q1: 打包后运行闪退
**原因**：缺少依赖或代码错误
**解决**：
```bash
# 先直接运行Python脚本测试
python main.py

# 查看打包日志
cat build/RedisVisualManager/warn-RedisVisualManager.txt
```

### Q2: 提示找不到模块
**原因**：`--add-data`路径错误
**解决**：
- macOS使用冒号`:`分隔
- Windows使用分号`;`分隔
- 确保源路径正确

### Q3: 打包文件过大
**原因**：包含了不必要的库
**解决**：
- 使用`--exclude-module`排除不需要的模块
- 使用虚拟环境，只安装必要依赖

### Q4: macOS提示"已损坏，无法打开"
**原因**：Gatekeeper安全机制
**解决**：
```bash
xattr -cr dist/RedisVisualManager.app
```

### Q5: Windows杀毒软件误报
**原因**：PyInstaller打包特征被误识别
**解决**：
- 添加杀毒软件白名单
- 使用代码签名证书
- 向杀毒软件厂商提交误报

## 九、自动化打包脚本

### macOS: `build_mac.sh`
```bash
#!/bin/bash
./build_mac.sh
```

### Windows: `build_win.bat`
```cmd
build_win.bat
```

两个脚本都会：
1. 清理旧的构建文件
2. 检查并安装依赖
3. 执行PyInstaller打包
4. 验证输出结果
