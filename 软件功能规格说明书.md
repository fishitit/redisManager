## 一、项目概述

### 1.1 软件名称
Redis可视化管理工具（Redis Visual Manager）

### 1.2 技术栈
| 组件 | 技术 | 说明 |
|------|------|------|
| 编程语言 | Python 3.6+ | 主语言 |
| GUI框架 | tkinter | Python内置GUI库 |
| Redis客户端 | redis-py | pip install redis |
| 架构模式 | MVC | Model-View-Controller |

### 1.3 运行环境
- Python 3.6 或以上版本
- 操作系统：Windows / macOS / Linux
- 依赖库：`redis>=4.0.0`

## 三、功能模块详细说明

### 3.1 连接管理模块

#### 3.1.1 多连接配置
**入口**：顶部状态栏 → "管理连接"按钮

**功能**：
1. **添加连接**
   - 点击"添加"按钮清空表单
   - 填写连接信息：
     - 别名：字符串，必填，唯一标识
     - 主机：字符串，必填，默认`127.0.0.1`
     - 端口：整数，必填，默认`6379`，范围1-65535
     - 密码：字符串，选填
     - 数据库：整数，必填，默认`0`，范围0-15
   - 点击"保存"按钮持久化到JSON文件

2. **编辑连接**
   - 左侧列表选择连接
   - 右侧表单自动填充
   - 修改后点击"保存"

3. **删除连接**
   - 左侧列表选择连接
   - 点击"删除"按钮，弹出确认对话框
   - 确认后删除并刷新列表

4. **测试连接**
   - 填写表单后点击"测试连接"
   - 尝试连接Redis并显示结果

5. **持久化存储**
   - 连接配置保存到 `connections.json`
   - JSON格式，UTF-8编码
   - 软件重启自动加载

**数据结构**：
```json
[
  {
    "alias": "本地开发",
    "host": "127.0.0.1",
    "port": 6379,
    "password": "",
    "db": 0
  }
]
```

#### 3.1.2 连接选择与切换
**入口**：顶部状态栏 → "选择连接"下拉框 + "连接"按钮

**功能**：
1. 下拉框显示所有已保存的连接别名
2. 选择连接后点击"连接"按钮
3. 或直接选择连接（触发自动连接）
4. 连接成功后状态栏显示：`已连接（别名）`（绿色文字）
5. 未连接状态显示：`未连接`（红色文字）

#### 3.1.3 断开连接
**入口**：顶部状态栏 → "断开连接"按钮

**功能**：
1. 点击后弹出确认对话框
2. 确认后关闭Redis连接
3. 状态恢复为"未连接"
4. 清空Key列表和详情

### 3.2 Key列表管理模块

#### 3.2.1 Key列表显示
**列定义**：
| 列名 | 宽度 | 对齐 | 说明 |
|------|------|------|------|
| Key | 300px | 左对齐 | Key名称，可拉伸 |
| 类型 | 80px | 居中 | 数据类型（string/hash/list/set/zset） |
| TTL(秒) | 100px | 右对齐 | 过期时间，可拉伸 |

**TTL显示格式**：
- 数字：剩余秒数（如`3600`）
- -1：显示`永不过期`
- -2：显示`不存在`

#### 3.2.2 搜索过滤
**过滤条件**：
1. **关键词过滤**：输入框，支持模糊匹配（前后加`*`）
2. **类型过滤**：下拉框，选项：`全部`、`string`、`hash`、`list`、`set`、`zset`

**操作按钮**：
- **搜索**：执行搜索并刷新列表
- **清空**：清空关键词和类型过滤，重置到第一页并刷新

#### 3.2.3 列排序
**功能**：点击列标题进行排序

**排序规则**：
- 第一次点击：升序（▲）
- 第二次点击：降序（▼）
- 切换列时重置为升序
- TTL列支持数字排序（`永不过期`>正数>`不存在`）
- 排序后标题显示方向箭头

#### 3.2.4 分页功能
**布局**：
```
左侧：[上一页] 第[输入框]页/共[X]页 [下一页] │ 分隔线 │ 总计:XXX个Key
右侧：每页显示: [下拉框▼]
```

**每页条数选项**：10、20、50（默认）、100、200、500

**功能**：
- 上一页/下一页按钮翻页
- 输入框直接输入页码，按Enter跳转
- 自动校正页码范围（1~总页数）
- 改变每页条数后重置到第1页

### 3.3 Key详情与编辑模块

#### 3.3.1 Key详情显示
**显示信息**：
- Key名称（蓝色文字）
- 数据类型
- TTL（过期时间）
- Value内容（根据类型格式化）

**Value格式化规则**：
| 类型 | 显示格式 |
|------|---------|
| string | 直接显示文本 |
| hash | 每行：`field: value` |
| list | 每行：`[索引] 元素` |
| set | 每行：`成员` |
| zset | 每行：`成员: 分数` |

#### 3.3.2 JSON校验与格式化
**按钮栏**（位于Value区域顶部）：
- 🔍 **JSON校验**：校验内容是否为有效JSON
- ✨ **格式化JSON**：将压缩JSON格式化为易读格式
- 📄 **原始内容**：恢复原始未格式化内容

**JSON校验**：
- 成功：状态标签显示`✓ JSON格式正确`（绿色），弹出成功提示
- 失败：状态标签显示`✗ JSON错误: 第X行`（红色），弹出详细错误信息

**JSON格式化**：
- 缩进：2空格
- 编码：`ensure_ascii=False`（保留中文）
- 键顺序：`sort_keys=False`（保持原顺序）
- 成功：状态标签显示`✓ JSON已格式化`（绿色），**不弹提示窗口**
- 失败：弹出错误提示

**原始内容**：
- 恢复未格式化的原始值
- 状态标签显示`✓ 已显示原始内容`（蓝色）

**状态标签**：
- 灰色：内容为空
- 绿色：JSON格式正确/已格式化
- 红色：JSON错误
- 蓝色：已显示原始内容

## 四、数据模型设计

### 4.1 ConnectionModel（连接管理模型）

**职责**：管理Redis连接配置的持久化

**方法**：
| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| load_connections | - | - | 从JSON文件加载配置 |
| save_connections | - | bool | 保存配置到JSON文件 |
| add_connection | alias, host, port, password, db | bool | 添加连接，检查别名唯一性 |
| update_connection | old_alias, alias, host, port, password, db | bool | 更新连接配置 |
| delete_connection | alias | bool | 删除连接配置 |
| get_connection | alias | dict/None | 获取指定连接配置 |
| get_all_connections | - | list | 获取所有连接配置 |

### 4.2 RedisModel（Redis操作模型）

**职责**：封装所有Redis操作

**属性**：
- `client`：redis.Redis实例
- `connected`：bool，连接状态
- `current_db`：int，当前数据库编号

**方法**：
| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| connect | host, port, password, db | (bool, str) | 连接Redis，返回(成功, 消息) |
| disconnect | - | - | 断开连接 |
| switch_db | db | (bool, str) | 切换数据库 |
| get_keys | pattern, page, page_size | (list, int) | 获取Key列表（分页） |
| get_keys_with_info | pattern, page, page_size | (list, int) | 获取Key列表（带类型和TTL） |
| get_key_type | key | str | 获取Key类型 |
| get_key_value | key | any | 获取Key值（按类型处理） |
| get_key_ttl | key | int | 获取过期时间 |
| set_key_ttl | key, ttl | bool | 设置过期时间 |
| delete_key | key | bool | 删除Key |
| rename_key | old_key, new_key | (bool, str) | 重命名Key |
| add_string_key | key, value, ttl | (bool, str) | 添加string类型 |
| add_hash_key | key, mapping, ttl | (bool, str) | 添加hash类型 |
| add_list_key | key, values, ttl | (bool, str) | 添加list类型 |
| add_set_key | key, members, ttl | (bool, str) | 添加set类型 |
| add_zset_key | key, mapping, ttl | (bool, str) | 添加zset类型 |
| update_string_value | key, value | (bool, str) | 更新string |
| update_hash_value | key, mapping | (bool, str) | 更新hash |
| update_list_value | key, values | (bool, str) | 更新list |
| update_set_value | key, members | (bool, str) | 更新set |
| update_zset_value | key, mapping | (bool, str) | 更新zset |
| get_server_info | - | dict | 获取服务器信息 |
| get_client_list | - | list | 获取客户端连接列表 |
| kill_client | addr | (bool, str) | 断开指定客户端 |

## 十一、注意事项

1. **首次运行**：需先通过"管理连接"添加Redis连接配置
2. **连接配置**：保存在`connections.json`，包含密码信息，注意文件权限
3. **大数据量**：大量Key时搜索和刷新可能需要时间
4. **清空操作**：FLUSHDB/FLUSHALL不可恢复，需二次确认
5. **CLIENT LIST**：需要Redis有CLIENT LIST命令权限
6. **编码**：所有文件使用UTF-8编码
7. **线程安全**：GUI操作在主线程执行，Redis操作同步执行
